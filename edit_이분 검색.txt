이분 검색(Binary Search)은 정렬된 데이터 구조에서 탐색 범위를 절반씩 줄여나가며 목표 값을 찾는 매우 효율적인 알고리즘입니다 (



 시간 복잡도).

코딩 및 알고리즘 테스트에서 이분 검색이 사용되는 주요 예시는 다음과 같습니다.
1. 기본적인 데이터 검색 (Target Value Search)
정렬된 배열이나 리스트에서 특정 숫자나 항목의 위치(인덱스)를 찾을 때 사용됩니다.

예시: 100만 개의 정렬된 숫자 데이터에서 578,231이라는 숫자가 어디에 있는지, 혹은 존재하는지 확인.
작동 방식: 중간값을 확인하고 목표 값이 더 크면 오른쪽, 작으면 왼쪽으로 범위를 좁힘.

2. 정렬된 배열에서의 Lower Bound / Upper Bound
정렬된 데이터에서 특정 값이 처음 나타나는 위치(Lower Bound)나, 특정 값보다 큰 첫 번째 위치(Upper Bound)를 찾을 때 사용합니다.
예시: [1, 2, 4, 4, 4, 6, 7]에서 '4'가 처음 등장하는 인덱스 찾기.
활용: 데이터의 빈도수 계산이나 중복된 데이터가 있는 정렬 배열에서 특정 조건의 범위를 찾을 때 사용됩니다. 
3. 파라메트릭 서치 (Parametric Search) - 최적화 문제
코딩 테스트에서 매우 자주 나오는 유형으로, '최적화 문제(최댓값/최솟값 찾기)'를 '결정 문제(Yes/No)'로 바꾸어 푸는 방식입니다. 답의 범위를 이분 검색으로 좁혀가며 최적의 답을 찾습니다. 
예시 (입국심사): 
명의 사람이 
개의 심사대를 통과하는 최소 시간 구하기.
이분 검색 활용: "X분 안에 모든 사람이 심사를 마칠 수 있는가?"를 Yes/No로 판단하여 X의 범위를 줄여나감.
예시 (예산 할당): 여러 부서의 예산 요청 중 총예산 내에서 최대 총액을 지원할 수 있는 상한액 찾기.
4. 회전된 정렬 배열에서의 검색
정렬된 배열을 한 번 회전시킨 배열(예: [4, 5, 6, 7, 0, 1, 2])에서 특정 값을 찾을 때 사용합니다. 절반을 나누었을 때 적어도 한쪽은 정렬되어 있다는 성질을 이용합니다. 
5. 정렬된 2차원 행렬(Matrix)에서의 검색
행과 열이 모두 정렬된 2차원 배열에서 특정 값을 찾을 때, 행렬을 하나의 긴 정렬된 배열로 간주하여 이분 검색을 적용할 수 있습니다.
